# Виправлення підключення SSH до сервера

## Проблема: Connection refused

Якщо `ssh root@188.245.228.175` не працює, спробуйте:

### Варіант 1: Інший порт SSH
```bash
# Деякі сервери використовують нестандартний порт
ssh -p 2222 root@188.245.228.175
# або
ssh -p 22022 root@188.245.228.175
```

### Варіант 2: Через CloudPanel
Згідно з логами сервера, там встановлено CloudPanel:
- URL: https://hosting.harbourtech.co/
- Або: https://188.245.228.175:8443

Можна підключитися через CloudPanel веб-інтерфейс та виконати команди через термінал там.

### Варіант 3: Перевірка доступності сервера
```bash
# Перевірте, чи сервер відповідає
ping 188.245.228.175

# Перевірте, які порти відкриті
nmap -p 22,2222,22022,8443 188.245.228.175
```

### Варіант 4: Через іншого користувача
```bash
# Спробуйте через користувача pro-part-agent
ssh pro-part-agent@188.245.228.175

# Або через ubuntu
ssh ubuntu@188.245.228.175
```

### Варіант 5: Через CloudPanel File Manager
1. Відкрийте https://hosting.harbourtech.co/ або https://188.245.228.175:8443
2. Увійдіть в CloudPanel
3. Знайдіть сайт agent.pro-part.online
4. Відкрийте File Manager
5. Перейдіть в директорію проекту
6. Виконайте команди через вбудований термінал CloudPanel

### Варіант 6: Якщо є доступ до CloudPanel
Можна виконати команди через CloudPanel CLI:
```bash
# Якщо є доступ до CloudPanel через веб-інтерфейс
# Використайте вбудований термінал або SSH через CloudPanel
```

1. Відкрити CloudPanel
2. Знайти сайт agent.pro-part.online
